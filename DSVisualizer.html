Data Structure Visualizer

The stack is a data structure that stores elements using the LIFO(Last In First Out) property.

Stacks have two main methods: push and pop. The push method pushes something onto the stack while the pop method pops something off the top of the stack.

Some real world use cases of a stack include: browsers storing previously visited URLs, managing open and closed brackets in programming languages, and the undo(ctrl+z) and redo(ctrl+y) functions in many applications.
Time complexity for Stack Methods:

Push: O(1)

Pop: O(1)
The Queue is a data structure that store elements using the FIFO(First In First Out) property.

Queues have two main methods: Enqueue and Dequeue. Enqueue puts something into the back of the queue and Dequeue takes something out of the front of the queue.

Some real world use cases of a Queue include: A queue of processes waiting for the CPU in an OS, a queue of packets in a router, and a queue of documents waiting to be printed by a printer.
Time complexity for Queue Methods:

Enqueue: O(1)

Dequeue: O(1)

Note that these time complexities are only possible if the Queue is implemented using a linked-list.
